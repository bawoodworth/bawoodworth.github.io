1. Intro to Velodyne

Overview of the Velodyne workflow and required software

From the Velodyne Manual v3.100:

Velodyne™ is an advanced explicit/implicit ﬁnite element framework incorporating state-ofthe-art techniques for analyzing the transient dynamic response of three-dimensional solids and structures developed by Corvid Technologies, Inc., to solve a variety of challenging problems using physics-based computational methodologies. The core of the code has its root in DYNA3D9having been developed at Lawrence Livermore National Laboratory since the early 1980’s. This robust and well validated tool was transitioned to commercial use through a technology transfer program and has been built upon and expanded to massively parallel computational operation by Corvid Technologies. Velodyne™ scales well across shared memory as well as distributed memory supercomputers with high-speed interconnect technology, and is envisioned as a high-ﬁdelity, system level continuum mechanics solver capable of handling hundreds of millions degrees of freedom in any given simulation.

The workflow for creating a Velodyne simulation typically requires the following steps: CAD, Meshing, Pre-Processing, Computation, and Post-Processing. For these lessons, we will focus on the last 3 steps: Pre- and Post-Processing, and Computation.

The software required for these steps, is as follows: LS-PrePost, VisIt, WinSCP and PuTTY. All of this software can be found for free online and there are links available under the Resources tab. Mac users will need to find comparable software for WinSCP.

Throughout these lessons, you will be provided with meshed models to begin with. While it would be useful to develop experience in meshing techniques, here we will focus on setting up simulations and interpreting the results.

LS-PrePost will be used for pre-processing the models and preparing them for simulation. Pre-processing allows us to get all of our ducks in a row, before sending our models off to Velodyne. In LS-PrePost we can organize our nodes into sets, name and manage our parts, and tie up any loose ends leftover from the meshing process.

Post-processing and visualization will be done using VisIt. VisIt is of course short for "visualize it" (I assume). In VisIt, we can parse out the data generated by Velodyne into contours on the model. Some of the types of contours that can be visualized are velocity-magnitude, von mises stress, pressure, strain, etc.

Once last piece of useful information is to know that Velodyne uses the centimeter-gram-seconds unit system. Wikipedia has a great table of obsure CGS unit names and their SI equivalents.
